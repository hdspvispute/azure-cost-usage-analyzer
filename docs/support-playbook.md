# Support Playbook

## Severity Model
- Sev-1: App unavailable for all users
- Sev-2: Core feature broken (cost or usage analysis)
- Sev-3: Non-critical bug or degraded behavior

## Response Targets
- Sev-1: immediate response
- Sev-2: same business day
- Sev-3: next sprint planning

## Triage Process
1. Capture user impact and scope.
2. Gather logs and recent deployment metadata.
3. Identify workaround and owner.
4. Apply fix and verify.
5. Record root cause and prevention action.

## Common Incidents
- Azure authentication failure
- No subscriptions returned
- Cost API empty/delayed data
- CI/CD deployment failure
